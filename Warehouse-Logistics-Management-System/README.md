# Logistics Integrated Management System
## Data Cleaning, Database Design & Operational Insights

### 📌 Project Overview
This project demonstrates a comprehensive end-to-end data engineering and business analysis workflow for a logistics system. It covers the entire lifecycle from processing raw messy data to designing a relational database schema and delivering actionable insights through interactive dashboards.

### 🛠️ Technologies Used
* **Data Processing:** Microsoft Excel (Power Query), Python (for cleaning).
* **Database Management:** MS Access, SQLite (SQL).
* **Data Modeling:** Star Schema (Fact & Dimension Tables), Data Normalization (1NF, 2NF, 3NF).
* **Visualization:** Power BI / Excel Dashboards.
* **Methodology:** Agile-Based Development.

### 📂 Project Structure & Workflow
1.  **Data Cleaning & ETL:**
    * Handled missing values and standardized date formats.
    * Calculated key metrics like `Lead Time` (Delivery Date - Order Date).
    * Transformed flat files into structured datasets.

2.  **Database Architecture:**
    * **Normalization:** Converted flat Excel data into a Relational Database to eliminate redundancy.
    * **Star Schema:** Implemented a robust model with a Fact table (Orders) and Dimension tables (Suppliers, Products, Shipping Methods).
    * **SQL Implementation:** Used SQL queries for data entry, automated reporting, and "Low Stock" alerts.

3.  **Operational Insights (KPIs):**
    * **Supplier Performance:** Analyzing total costs and delivery reliability for major suppliers (Samsung, DHL, Amazon).
    * **Shipping Efficiency:** Monitoring the ratio of Delayed vs. Delivered orders.
    * **Inventory Management:** Tracking product categories (Electronics, Construction Materials, etc.) and stock levels.

### 📊 Key Findings
* Identification of suppliers with the highest average lead times.
* Cost distribution analysis across different logistics categories.
* Visualization of shipping status trends to optimize supply chain operations.
